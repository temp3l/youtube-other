# YouTube Short Story Quality Analysis

## 1. Executive Summary

- Overall quality: Episode 021 Shorts are weak as standalone YouTube Shorts. They preserve a few labels and the final line, but not the story engine.
- Main confirmed root cause: the Short prompt for Episode 021 only receives three retained source beats, not the full story or a real beat map.
- German output was accepted even though it is abstract, repetitive, awkward as spoken German, and poorly localized.
- English output in `debug/stories-rewrite-short-en.response-text.json` was generated by the pipeline but recorded as failed in the manifest.
- The selected beats are structurally poor: opening location, one middle “checking” sentence, final folklore consequence.
- Essential horror beats are omitted: tapping at night, 2:11, camera footage, pale figure, synchronized breathing, wardrobe opening, creature leaving, expression-copying reveal.
- Current output behaves like a thematic recap or synopsis, not a retention-optimized mini-story.
- Validation is mostly structural: word count, duration, hook timing, contract phrase presence, unsupported facts, some synopsis markers.
- Validation does not adequately block abstraction, semantic repetition, weak causality, low event density, weak hook, poor German fluency, or unlocalized metadata.
- Metadata/localization is inconsistent: German short metadata contains English title, thumbnail text, and bridge text.
- This is systemic risk, not isolated: Episode 010 contains planning/editorial language; Episode 014 shows the pipeline can produce a good Short when source selection and prose align.
- Fix first: replace sentence-level retention with a Short-specific event plan requiring hook, evidence, escalation, reversal, and sting.

## 3. Current Generation Pipeline

Observed flow:

```text
Validated full story artifact
  -> canonical facts / StoryIR
  -> sentence splitting into source beats
  -> heuristic retained-beat selection
  -> Short Adaptation Contract
  -> prompt compilation
  -> OpenAI structured short narration request
  -> validation
  -> repair or regeneration
  -> markdown/json persistence
  -> compatibility rendering / metadata-like fields
```

Key observed stages:

- Full parent resolution: `resolveShortRewriteParent` in `short-rewrite.service.ts`.
- Beat extraction: `buildShortSourceExtraction` in `short-adaptation-contract.ts`.
- Beat selection: `selectRetainedBeatIds` in `short-adaptation-contract.ts`.
- Contract construction: `buildShortAdaptationContract`.
- Prompt rendering: `compileShortStoryPrompt` and short contract block in `story-prompt-compiler.ts`.
- Generation request: `generateShortNarration` flow in `short-rewrite.service.ts`.
- Validation: `validateShortNarrationArtifact` in `generated-story-validator.ts`.
- Markdown output: `buildShortRewriteMarkdown` in `short-rewrite.renderer.ts`.

The critical behavior is at `story-prompt-compiler.ts:380-384`: the prompt passes only retained beats inside `<SHORT_ADAPTATION_SOURCE>`. For Episode 021, those are only `b01`, `b49`/`b55`, and `b98`/`b110`.

## 5. Main Problems

### 5.1 Weak Hook Selection

Evidence: Full story opens with “no balcony,” “no tree,” “no ladder,” and tapping at night. The Short prompt retained only the first location sentence (`b01`) and omitted `b02-b05`. In `de-short.json`, `b05` is explicitly removed.

Viewer impact: first seconds do not communicate impossibility or danger.

Production impact: visual generation repeats farmhouse/window instead of showing impossible evidence.

Likely root cause: `selectRetainedBeatIds` prioritizes StoryIR phrases and first sentence, not the strongest Short premise.

Confidence: confirmed.

### 5.2 Missing Mini-Arc

Evidence: Episode 021 Short has setup, verification, final folklore. It lacks escalation and reversal.

Viewer impact: no complete mini-story; ending feels pasted on.

Production impact: no sequence of visual beats.

Likely root cause: prompt asks for narrative functions but supplies insufficient source beats.

Confidence: confirmed.

### 5.3 Excessive Abstraction

Evidence: English: “fear stayed vague,” “more exact,” “attention kept returning.” German: “Punkt, an dem alles hängen blieb,” “Und damit endet die Geschichte…”

Viewer impact: reduced comprehension, tension, visualization, and authenticity.

Production impact: abstract sentences require filler footage.

Likely root cause: overconstrained source package forces the model to inflate three facts into 145-170 words.

Confidence: high.

### 5.4 Semantic Repetition

Evidence: English repeats checking/window/pattern/looking across most sentences. German repeats Fenster/Scheibe/Obergeschoss plus checking.

Viewer impact: low information density; likely swipe after 10-20 seconds.

Production impact: repetitive visuals.

Likely root cause: retained beats are all window/checking/final-window, and validator lacks semantic repetition checks.

Confidence: confirmed.

### 5.5 Missing Critical Plot Events

Evidence: `de-short.json` selected only `b01`, `b55`, `b110`; removed beats include 2:11, camera, figure, wardrobe, inside/outside reversal.

Viewer impact: final line lacks causal force.

Production impact: strongest visual beats unavailable to downstream stages.

Likely root cause: StoryIR misidentifies central threat/rule/climax as location plus final folklore, not the actual reversal.

Confidence: confirmed.

### 5.6 Weak Causality

Evidence: The Shorts say the window matters but never show why. The final man asks people to look directly, but the eye-contact rule is omitted.

Viewer impact: viewer understands “creepy window” but not the threat mechanics.

Production impact: visual plan cannot represent rule-based escalation.

Likely root cause: preservation contract checks phrase presence, not causal prerequisites.

Confidence: high.

### 5.7 German Localization Problems

Evidence:

- “das Fenster war das, über das” is clumsy.
- “der Punkt, an dem alles hängen blieb” is abstract idiom misuse.
- “wegen dem” is unsuitable in polished narration.
- “ein Blick daran kleben blieb” sounds literal/machine-like.
- “Petra Marlow” repeated instead of natural “Petra” after introduction.
- “Petra” is normally female-coded in German, while the character is male in story context.

Viewer impact: sounds generated, not native spoken horror.

Production impact: narration quality and audience trust drop.

Likely root cause: locale validator only checks broad language tokens and leakage, not idiomatic fluency or gender/name appropriateness.

Confidence: high.

### 5.8 English Narration Problems

Evidence: “more exact,” “attention kept returning,” “one window, one height,” and “the story has lasted because” sound like analysis or trailer copy.

Viewer impact: less natural aloud; feels like an AI summary.

Production impact: fewer concrete performance cues.

Likely root cause: model expands an abstract retained beat, plus previous invalid result leaked the same phrasing into regeneration context.

Confidence: high.

### 5.9 Duration and Pacing Problems

Evidence: German accepted at 165 words. At configured German 195 WPM it is about 50.8 seconds, below the 55-second duration window, while JSON estimates only 175/180 WPM. English is 156 words, about 45.7 seconds at 205 WPM.

Viewer impact: either rushed narration or too little story for 55-65 seconds.

Production impact: mismatch between script duration, pauses, and visual pacing.

Likely root cause: duration checks use target WPM in validation, but persisted estimates are fixed 175/180; instructions mention 175-180 even for German renderer.

Confidence: confirmed.

### 5.10 Visual Production Problems

Evidence: Current Short has mostly window/checking/looking. Full story offers many concrete visuals: frozen ground, condensation marks, 2:11 clock, pale figure, mirror, ceiling handprints, wardrobe, closed glass, camera movement.

Viewer impact: fewer pattern interrupts.

Production impact: image generation likely repeats farmhouse/window shots.

Likely root cause: Short selection strips visual beats before visual planning can use them.

Confidence: high.

### 5.11 Metadata Problems

Evidence: `de/short/021...json` has `title: "Something Was Watching From the Bedroom Window"`, `thumbnailText: "Something Was Watching From"`, and `fullVideoBridge: "Watch the full episode..."`. `de/short/metadata.json` mirrors this. Renderer also uses English “Audio Generation Instructions” headings.

Viewer impact: inconsistent localized package.

Production impact: German publication artifacts need manual cleanup.

Likely root cause: `analyzeGeneratedPayload` derives title/thumbnail/bridge from parent title and hardcoded English strings.

Confidence: confirmed.

### 5.12 Validation Gaps

Evidence: German artifact validation has `warnings: []` despite weak prose, abstraction, repetition, metadata issues, and poor mini-arc.

Viewer impact: bad scripts can pass.

Production impact: production proceeds with low-quality accepted assets.

Likely root cause: `validateShortNarrationArtifact` does not test abstract ratio, event density, causal completeness, visual beat density, locale naturalness, or metadata localization.

Confidence: confirmed.

## 9. Recommended Content-Preservation Rules

### Mandatory

- Premise: impossible second-floor window.
- Central threat: pale figure/tapping connected to the window.
- Protagonist action: Petra verifies or records evidence.
- Escalation event: camera, 2:11, breathing, or ceiling/wardrobe.
- Primary reveal: it was already inside and leaving.
- Ending consequence: man tapping from inside asking for direct eye contact.
- Chronology: outside tapping before inside reversal.
- Story rule: direct looking/eye contact matters.

### Strongly preferred

- No balcony/tree/ladder.
- Condensation marks and no footprints.
- Exact time 2:11.
- Camera recording.
- Wardrobe opens from inside.
- Creature passes through closed glass.
- Facial-expression copy.

### Optional

- Brother’s care reason.
- Barn nest of photographs.
- Burning photos.
- Police search.
- Failed attempt to leave.

### Usually removable

- General atmosphere.
- Repeated checking.
- Abstract commentary about patterns.
- Meta explanation of why legend lasted.
- Broad “people say” framing before final line.

- Decide whether the Short is a teaser or a complete standalone story.
- Review localized names and gender coding.
- Confirm thumbnail text is short, localized, and non-technical.
- Listen aloud for German and English narration rhythm.
- Check whether final line would make sense to a viewer with no prior context.

## 11. Prompt-Level Recommendations

| Current failure mode | Proposed instruction | Expected behavior | Possible negative side effect |
| -------------------- | -------------------- | ----------------- | ----------------------------- |
| Abstract recap | “Every sentence must either show a physical event, a piece of evidence, a decision, or the final consequence.” | Less thematic filler | May reduce atmospheric prose |
| Weak hook | “Open with the impossible event, not the setting.” | Stronger first second | Could overuse shock openings |
| Unearned ending | “Do not use the final folklore line unless the Short has established why inside/outside and direct looking matter.” | Better causality | Some endings need compression |

| Over-preservation | “Preserve narrative function, not exact sentence wording, except exact written messages.” | Better adaptation | Needs stronger fact validator |

## 12. Pipeline-Level Recommendations

- Source-story analysis: identify “Short-worthy” hook candidates and visual/reversal beats from full story.
- Beat extraction: store events with role labels instead of raw sentence beats.
- Short planning: generate/select a compact event outline before prose.
- Prose generation: consume the outline plus allowed source facts, not just three sentences.
- Localization: adapt German independently from the event plan, not from English phrasing or literal retained sentences.
- Validation: add semantic checks for mini-arc, event density, abstraction, repetition, reveal prerequisites, and locale fluency.
- Retry logic: when abstraction/repetition fails, regenerate from the event plan; do not include failed abstract prose unless needed diagnostically.
- Metadata generation: localize title, thumbnail, descriptions, tags, hashtags, and bridge copy for the target locale.
