## Locale And `sp` Findings

- Canonical TypeScript locale/language sets inspected:
  - `packages/story-localization/src/story-localization.types.ts`: `["en","de","es","fr","pt"]`.
  - `packages/shared/src/episode-filesystem.ts`: `["en","de","es","fr","pt"]`.
  - `packages/story-localization/src/language-profiles.ts`: Spanish is `es` with locale `es-419`.
  - Short rewrite schemas enumerate `en,de,es,fr,pt`.
- Relevant `sp` occurrences found in inspected search output were not canonical locale definitions. The repository does contain unrelated substrings such as `response`, `inspect`, `Spanish`, and env names with `SPANISH`.
- Required implementation check: add a focused search/test gate that validates exact locale tokens in CLI input, manifests, cache entries, batch items, and artifact paths; reject `sp` or migrate it to `es` only through a documented compatibility migration.
