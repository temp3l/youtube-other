## visual retention

Recommended architecture: hybrid explicit scene-to-shot architecture. Major chain: characterization → schemas → paths/config/catalog/focal metadata → deterministic planner → validators → FFmpeg builders → shot renderer/cache → CLI preview → pipeline integrations → telemetry.

Parallel tasks: 04/05/06, 09 with planner work after schemas, 11 with planner work after catalog, 14/17 after cache.

Expected Shorts output: 5-12 source images generating 15-35 shots, with roughly 55-70% avoided image-generation calls.

Major risks: renderer contract churn, cache explosion, visual repetition despite crops, caption/evidence collisions, Dark Truth path drift, FFmpeg filter fragility.

Deprecated components identified: one-scene-one-image assumptions, scene-only clip IDs/manifests, unused Shorts motion metadata, raw FFmpeg string growth, scene-only visual plans.

GPT-5.5 Medium is sufficient for most tasks; High may help for Tasks 07, 11, 12, 15, and 16.

## batch

• Use these from the CLI:
content-ideas/content/dark-truth-episodes-optimized/020-teke-teke-en-full-optimized.md

npm run mediaforge -- stories pipeline \
 --episode 020-teke-teke \
 --locales en,de \
 --formats full,short \
 --dry-run \
 --batch-mode batch \
 --json

That plans a workflow and prints the planned manifest. Useful flags:

- --cost-estimate
- --batch-mode sync|batch|hybrid
- --resume <workflow-id> to reuse an existing manifest if present
- --output-root <path> to point at a different workspace

To read back a persisted workflow:

npm run mediaforge -- stories pipeline status \
 --episode 009-the-christmas-doll \
 --workflow wf_009-the-christmas-doll_20260701T000000Z_deadbeef \
 --json

Human-readable status is the same command without --json.
